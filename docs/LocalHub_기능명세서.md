# LocalHub 기능 명세서

> 현재 구현 기준으로 작성한 기능 명세서입니다.
> 작성일: 2026-07-16

## 1. 서비스 개요

LocalHub는 Vue 3 + Vite 기반의 서울 레포츠 장소 추천 웹앱입니다.
사용자는 서울의 레포츠 장소를 탐색하고, 커뮤니티에서 의견을 나누며, 방문한 자치구를 기록해 도장처럼 모아갈 수 있습니다.

- 주요 대상: 서울에서 운동하고 싶은 사용자
- 핵심 기능: 장소 탐색, 커뮤니티, 도장 깨기, 자연어 챗봇 추천
- 데이터 저장 방식: 브라우저 localStorage 기반

---

## 2. 현재 구현된 기능

### 2.1 홈 화면

- 상단 네비게이션을 통해 홈, 장소 목록, 커뮤니티, 도장 깨기 페이지로 이동 가능
- 히어로 섹션과 CTA 버튼으로 주요 기능 진입 가능
- 최근 작성된 커뮤니티 게시글 3개를 홈 화면에 미리 보여줌
- 커뮤니티로 이동하는 링크 제공

### 2.2 장소 목록 화면

- /places 경로에서 서울 레포츠 장소 목록 표시
- public/data/서울_레포츠.json 데이터를 불러와 목록 렌더링
- 구 단위 선택 필터를 통해 특정 자치구 장소만 조회 가능
- 장소 카드에 이미지, 제목, 주소, 자치구 태그 표시
- 챗봇 영역과 함께 배치되어 자연어로 구 기반 추천 요청 가능

### 2.3 커뮤니티 화면

- /community 경로에서 게시글 목록 조회 가능
- 제목, 내용, 태그 기반 검색 지원
- 최신순 / 조회수순 정렬 지원
- 페이지네이션으로 게시글 목록 분할 표시
- 글쓰기 모달을 통해 새 게시글 작성 가능
- 제목, 내용, 비밀번호, 태그 입력 및 유효성 검사 지원
- 작성한 게시글은 로컬 저장소에 저장되어 새로고침 후 유지

### 2.4 커뮤니티 상세 화면

- /community/:id 경로에서 게시글 상세 조회 가능
- 조회수 증가 처리
- 수정 및 삭제 기능 제공
- 수정/삭제 시 비밀번호 확인 절차 포함
- 좋아요 및 북마크 토글 기능 제공
- 좋아요/북마크 상태는 localStorage에 저장됨

### 2.5 도장 깨기 화면

- /stamp 경로에서 서울 25개 자치구 방문 기록 입력 가능
- 각 자치구별 방문 횟수 입력 및 +/- 버튼 조정 가능
- Chart.js 기반 버블 차트로 방문 현황 시각화
- 방문 횟수에 따라 시각적 단계 구분 제공
- 입력값은 localStorage에 저장되어 새로고침 후 유지
- 전체 초기화 기능 제공

### 2.6 챗봇 기능

- 장소 추천 챗봇이 /places 화면에 포함됨
- 사용자가 구 이름을 자연어로 입력하면 해당 구의 추천 장소를 응답
- OpenRouter API 키가 있으면 AI 기반 추출, 없으면 로컬 fallback 방식 사용
- 대화 기록은 localStorage에 저장되어 새로고침 후 복원 가능

## 3. 라우팅 구조

| 경로 | 화면 | 설명 |
|---|---|---|
| / | HomeView | 서비스 소개 및 주요 기능 진입 |
| /places | PlaceListView | 서울 레포츠 장소 목록 및 챗봇 |
| /community | CommunityView | 커뮤니티 목록 및 글쓰기 |
| /community/:id | CommunityDetailView | 게시글 상세 및 수정/삭제 |
| /stamp | StampView | 서울 자치구 방문 기록 및 차트 |

## 4. 데이터 소스

### 4.1 장소 데이터

- 데이터 파일: public/data/서울_레포츠.json
- 사용 목적: 서울 레포츠 장소 목록 표시
- 현재 화면에서는 장소명, 주소, 이미지, 자치구 정보 중심으로 표시

### 4.2 커뮤니티 데이터

- 저장 위치: localStorage
- 키: localhub:v1:posts
- 저장 정보: 제목, 내용, 비밀번호, 태그, 조회수, 좋아요 수, 생성/수정 시간

### 4.3 좋아요/북마크 데이터

- 저장 위치: localStorage
- 키: localhub:v1:likes, localhub:v1:bookmarks

### 4.4 챗봇 데이터

- 저장 위치: localStorage
- 키: localhub-chatbot-messages

### 4.5 도장 데이터

- 저장 위치: localStorage
- 키: stampRallyData

## 5. 주요 구현 파일

- src/views/HomeView.vue: 홈 화면 UI
- src/views/PlaceListView.vue: 장소 목록 및 구 필터 UI
- src/views/CommunityView.vue: 커뮤니티 목록, 검색, 글쓰기
- src/views/CommunityDetailView.vue: 게시글 상세, 수정, 삭제, 좋아요/북마크
- src/views/StampView.vue: 도장 깨기 차트 및 방문 기록
- src/components/chatbot/Chatbot.vue: 챗봇 UI 및 메시지 처리
- src/api/openRouter.js: 구 단위 추출 및 OpenRouter 연동
- src/services/recommendService.js: 장소 추천 로직
- src/servies/postService.js: 커뮤니티 CRUD 및 로컬 저장 로직
- src/router/index.js: 라우팅 설정

## 6. 제약 사항

- 서버 없이 프론트엔드만으로 동작하는 로컬 프로토타입 형태임
- 데이터는 브라우저 로컬 저장소에만 저장되므로 다른 기기나 브라우저에서는 공유되지 않음
- 게시글 비밀번호는 평문으로 저장되므로 실제 서비스 수준의 보안은 적용되지 않음
- 챗봇은 기본적으로 지역명 추출 기반이며, OpenRouter API는 선택적으로 사용됨

- 챗봇의 더 정교한 자연어 추천

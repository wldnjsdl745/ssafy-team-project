# LocalHub 화면·데이터·localStorage 구조 및 기능 명세서

> **LOCALHUB · MVP SPECIFICATION**

서울 지역 레포츠 장소 탐색, 익명 커뮤니티, 방문 도장 깨기, 자연어 챗봇, 활동 대시보드를 Vue 3 + Vite 정적 SPA 환경에서 구현하기 위한 상세 설계 문서입니다.

- **선정 권역:** 서울
- **타깃:** 운동 마니아
- **저장소:** JSON + localStorage
- **배포:** Netlify

---

## 1. 문서 개요

### 1.1 핵심 문제

다양한 운동 장소를 탐색하고 방문 성취를 기록하기 어렵다.

### 1.2 핵심 가치

장소 탐색, 방문 체크, 달성률 확인을 한 화면 흐름으로 제공한다.

### 1.3 기술 구조

- Vue 3
- Vite
- Vue Router
- Pinia 선택
- Chart.js
- 지도 라이브러리

### 1.4 성공 기준

- 필수 요구사항 100% 구현
- 선택 기능 1개 이상 구현
- 배포 URL 정상 동작

> **중요 제약:** 별도 백엔드가 없으므로 게시글, 방문 기록, 채팅 기록은 현재 브라우저에만 저장됩니다. OpenAI API 키 또한 프론트 빌드 결과에서 완전히 은닉할 수 없습니다.

## 4. 화면 구조 설계

### 4.1 공통 레이아웃

- 상단: 로고, 장소 탐색, 도장 깨기, 커뮤니티, 대시보드
- 모바일: 햄버거 메뉴 또는 하단 내비게이션
- 우측 하단: 챗봇 플로팅 버튼
- 전역 상태: 토스트, 확인 모달, 로딩, 빈 상태

### 4.2 홈

- 서울 레포츠 탐색 CTA
- 추천 장소 4~6개
- 방문 완료 수 / 전체 장소 수
- 최근 게시글 3개
- 도장 깨기 진행률 바

### 4.3 장소 목록

- 검색창: 장소명, 주소, 운동 종목
- 필터: 자치구, 운동 종류, 방문 여부
- 정렬: 이름순, 미방문 우선, 최근 방문순
- 목록/지도 보기 전환
- 카드: 장소명, 종목, 주소, 방문 상태

### 4.4 장소 상세

- 장소 기본 정보와 운영 정보
- 지도 위치 및 길찾기 외부 링크
- 방문 완료/취소 버튼
- 방문 일시 표시
- 관련 커뮤니티 글 연결

### 4.5 도장 깨기

- 전체 진행률
- 운동 종목별 달성률
- 방문 완료 장소 그리드
- 미방문 추천 장소
- 최근 방문 기록

### 4.6 커뮤니티

- 게시글 목록·상세·작성·수정·삭제
- 제목/내용 검색
- 최신순·조회수순 정렬
- 태그·좋아요·북마크는 선택 구현
- 로컬 저장 한계 안내 문구 표시

### 4.7 대시보드

- 방문 장소 수 KPI
- 전체 달성률 도넛 차트
- 운동 종류별 달성률 막대 차트
- 자치구별 방문 분포
- 커뮤니티 작성·조회 통계

### 4.8 챗봇 패널

- 열기/닫기, 전체 화면 전환
- 추천 질문 버튼
- 사용자·챗봇 메시지 구분
- 관련 장소/게시글 카드 응답

## 5. 데이터 구조 설계

## 5.1 데이터 출처 및 라이선스

| 항목 | 내용 |
|---|---|
| 제공 기관 | 한국관광공사 |
| 데이터명 | 국문 관광정보 서비스(TourAPI 4.0) |
| 사용 파일 | `서울_레포츠.json` |
| 콘텐츠 유형 | 레포츠 |
| 수집 지역 | 서울(SEL) |
| 사용 건수 | 126건 |
| 공공데이터 포털 | 한국관광공사 Tour API v4 |
| 라이선스 | 공공누리 제3유형(출처 표시 + 변경 금지) |

이 서비스는 한국관광공사 Tour API(TourAPI 4.0)의 `서울_레포츠.json` 데이터 126건을 활용하였습니다.

- **출처:** 한국관광공사 공공데이터 포털 원본 API
- **라이선스:** 공공누리 제3유형

> **데이터 사용 원칙:** 원본 JSON의 내용은 직접 수정하지 않습니다. 화면 구현에 필요한 검색·필터·표시 처리는 애플리케이션 실행 중 필드 매핑 방식으로 수행하며, 원본 파일 자체를 수정한 형태로 재배포하지 않습니다.

## 5.2 레포츠 장소 데이터 표준 모델

`서울_레포츠.json`의 원본 필드는 유지하고, 화면에서는 원본 데이터를 직접 변경하지 않는 런타임 매핑 방식으로 아래 표준 모델을 사용합니다.

```json
{
  "id": "seoul-place-001",
  "name": "한강공원 러닝코스",
  "category": "러닝",
  "district": "영등포구",
  "address": "서울특별시 영등포구 ...",
  "latitude": 37.528,
  "longitude": 126.934,
  "description": "한강변을 따라 달릴 수 있는 코스",
  "operatingHours": "상시 개방",
  "fee": "무료",
  "phone": "02-0000-0000",
  "homepageUrl": "https://...",
  "imageUrl": "./assets/...",
  "source": "한국관광공사 TourAPI 4.0 / 서울_레포츠.json",
  "updatedAt": "2026-07-14"
}
```

| 필드 | 타입 | 필수 | 설명 | 기본 처리 |
|---|---|---:|---|---|
| `id` | String | Y | 장소 고유 식별자 | 원본 ID가 없으면 지역+순번으로 생성 |
| `name` | String | Y | 장소명 | 빈 값은 목록에서 제외 |
| `category` | String | Y | 러닝, 수영, 클라이밍 등 운동 종류 | 분류 불가 시 `기타` |
| `district` | String | Y | 서울 자치구 | 주소에서 추출 가능 |
| `address` | String | Y | 도로명 또는 지번 주소 | 지도 링크 생성에 사용 |
| `latitude` | Number | 지도 사용 시 Y | 지도 마커 좌표 | 좌표가 없으면 지도 마커 제외 |
| `longitude` | Number | 지도 사용 시 Y | 지도 마커 좌표 | 좌표가 없으면 지도 마커 제외 |
| `description` | String | N | 장소 소개 | 없으면 기본 안내 문구 |
| `operatingHours` | String | N | 운영 시간 | `정보 없음` 표시 |
| `fee` | String | N | 이용 요금 | `정보 없음` 표시 |

## 6. localStorage 구조 설계

> **키 네이밍 규칙:** 충돌 방지를 위해 모든 키에 `localhub:` 접두사를 사용하고, 데이터 구조 변경을 고려해 버전 `v1`을 포함합니다.

| 키 | 값 타입 | 저장 내용 | 갱신 시점 |
|---|---|---|---|
| `localhub:v1:posts` | `Array<Post>` | 익명 커뮤니티 게시글 전체 | 작성·수정·삭제·조회수 변경 |
| `localhub:v1:visits` | `Array<Visit>` | 장소별 방문 완료 기록 | 방문 체크·취소·메모 변경 |
| `localhub:v1:chatMessages` | `Array<ChatMessage>` | 챗봇 대화 히스토리 | 질문·답변 생성·대화 초기화 |
| `localhub:v1:bookmarks` | `Array<String>` | 북마크한 게시글 ID | 북마크 토글 |
| `localhub:v1:likes` | `Array<String>` | 좋아요를 누른 게시글 ID | 좋아요 토글 |
| `localhub:v1:preferences` | Object | 목록 보기 방식, 최근 필터, 챗봇 열림 상태 | 사용자 UI 설정 변경 |
| `localhub:v1:schemaVersion` | String | 저장 데이터 버전 | 초기 실행·마이그레이션 |

### 6.1 저장 유틸리티 권장 인터페이스

```js
const STORAGE_KEYS = {
  posts: 'localhub:v1:posts',
  visits: 'localhub:v1:visits',
  chatMessages: 'localhub:v1:chatMessages',
  bookmarks: 'localhub:v1:bookmarks',
  likes: 'localhub:v1:likes',
  preferences: 'localhub:v1:preferences'
}

function loadStorage(key, fallback) {
  try {
    const raw = localStorage.getItem(key)
    return raw ? JSON.parse(raw) : fallback
  } catch (error) {
    console.error('localStorage parse error:', error)
    return fallback
  }
}

function saveStorage(key, value) {
  localStorage.setItem(key, JSON.stringify(value))
}
```

> **비밀번호 저장 주의:** RFP 요구에 따라 게시글 비밀번호를 localStorage에 저장하지만, 평문 저장은 보안 기능이 아닙니다. 화면에 **“교육용 로컬 프로토타입”**임을 명시해야 합니다.

## 7. 기능 명세서

## 7.1 레포츠 장소 조회

| ID | 기능명 | 입력/트리거 | 처리 | 출력/예외 | 우선순위 |
|---|---|---|---|---|---|
| PLC-01 | 장소 목록 로드 | `/places` 진입 | 제공 JSON fetch → 표준 모델 변환 → 목록 표시 | 실패 시 오류 안내와 재시도 버튼 | Must |
| PLC-02 | 장소 검색 | 검색어 입력 | 장소명·주소·운동 종목을 부분 일치 검색 | 결과 수 표시, 0건이면 빈 상태 | Must |
| PLC-03 | 장소 필터 | 자치구·종목·방문 여부 선택 | 다중 조건 AND 적용 | 필터 초기화 제공 | Must |
| PLC-04 | 장소 상세 조회 | 장소 카드 클릭 | ID로 장소 데이터 조회 | 존재하지 않으면 404 대체 화면 | Must |
| PLC-05 | 지도 마커 표시 | 지도 보기 선택 | 좌표가 있는 장소를 마커로 표시 | 마커 클릭 시 요약 팝업 | Should |

## 7.2 도장 깨기

| ID | 기능명 | 입력/트리거 | 처리 | 출력/예외 | 우선순위 |
|---|---|---|---|---|---|
| STP-01 | 방문 완료 등록 | 장소 상세의 방문 완료 버튼 | `visits`에 `placeId·visitedAt` 저장 | 버튼 상태 변경, 토스트, 진행률 즉시 갱신 | Must |
| STP-02 | 방문 취소 | 방문 완료 상태에서 취소 | 해당 방문 기록 삭제 또는 `visited=false` | 확인 모달 후 진행률 재계산 | Must |
| STP-03 | 전체 달성률 계산 | 홈·도장 깨기·대시보드 진입 | 방문 수 ÷ 전체 장소 수 × 100 | 0~100%로 표시 | Must |
| STP-04 | 종목별 달성률 | 대시보드 진입 | 종목별 전체 장소와 방문 장소 집계 | 막대 차트 또는 진행률 목록 | Selected |

## 9. 챗봇 데이터 처리 설계

### 9.1 처리 흐름

```text
사용자 질문
자연어 입력
    ↓
로컬 검색
장소·게시글 후보 추출
    ↓
프롬프트 구성
관련 데이터만 축약
    ↓
OpenAI 호출
응답 생성
    ↓
카드 렌더링
장소·게시글 링크
```

### 9.2 권장 처리 방식

1. 질문에서 자치구, 운동 종류, 장소명과 같은 키워드를 우선 추출합니다.
2. 전체 JSON을 그대로 API에 보내지 않고, 프론트에서 필터링한 상위 5~10개 후보만 전달합니다.
3. 게시글도 제목·내용 일부·태그만 축약하여 전달합니다.
4. 응답은 장소 또는 게시글 ID를 함께 반환하도록 프롬프트를 구성해 화면 링크와 연결합니다.

> **API 키 관리:** `VITE_OPENAI_API_KEY`는 브라우저 번들에서 확인될 수 있습니다. 프로젝트 전용 제한 키, 낮은 결제 한도, 제출 후 폐기 정책을 적용해야 합니다.

## 10. 데이터 시각화 대시보드 설계

| 위젯 | 계산식·데이터 | 표현 방식 | 빈 데이터 처리 |
|---|---|---|---|
| 방문 장소 수 | `visits.filter(v => v.visited).length` | KPI 카드 | `0개 표시 + 장소 탐색 CTA` |
| 전체 달성률 | 방문 장소 수 ÷ 전체 장소 수 × 100 | 도넛 차트 | 0% 상태 설명 |
| 종목별 달성률 | 종목별 방문 수 ÷ 종목별 전체 수 | 가로 막대 차트 | 방문 기록 없는 종목도 0으로 표시 |
| 자치구별 방문 분포 | 방문 장소를 `district`로 그룹화 | 막대 또는 레이더 차트 | 상위 자치구 없음 표시 |
| 최근 방문 | `visitedAt` 내림차순 상위 5개 | 타임라인 | 첫 방문 유도 메시지 |
| 커뮤니티 활동 | 게시글 수, 총 조회수, 좋아요 수 | 보조 KPI 카드 | 게시글 작성 CTA |

> **차트 갱신:** 방문 체크·취소 또는 게시글 변경 직후 Vue의 반응형 상태를 갱신하고, 동일 데이터를 localStorage에 저장합니다. 차트 컴포넌트는 계산된 통계 값을 props로 전달받습니다.
